# برنامه افزودن forecast / previous / actual

## رویکرد انتخاب‌شده
فقط اگر استخراج این فیلدها از HTML به‌صورت قابل‌اعتماد ممکن باشد، آن‌ها را وارد کد می‌کنیم.

## دلیل
در Forex Factory این فیلدها معمولاً در ستون‌های جداگانه جدول دیده می‌شوند، اما:
- ساختار HTML ممکن است تغییر کند
- بعضی سطرها ممکن است این مقدارها را نداشته باشند
- بعضی رویدادها قبل از انتشار actual ندارند

## معیار اضافه شدن به کد
این فیلدها فقط وقتی باید وارد parser شوند که:
- selector مشخص و پایدار داشته باشند
- روی چند نمونه سطر قابل‌اتکا باشند
- باعث خراب شدن parser اصلی نشوند

## اگر قابل‌اعتماد بود
فیلدهای جدید پیشنهادی در مدل Event:
- `actual`
- `forecast`
- `previous`

و در آینده:
- `revision`
- `source_url`

## استفاده در موتور تحلیل
اگر `forecast` و `previous` موجود باشند:
- می‌توان bias را قبل از خبر دقیق‌تر کرد
- می‌توان اهمیت تغییر مورد انتظار را بهتر سنجید

اگر `actual` بعد از خبر موجود باشد:
- می‌توان post-release reaction model ساخت
- divergence analysis اجرا کرد

## پیشنهاد مرحله بعد
در مرحله بعد باید HTML واقعی جدول با همین هدف بررسی شود.
